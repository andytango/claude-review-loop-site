claude-review-loop

A Claude Code plugin that reviews your changes with adversarial AI agents, then fixes what you approve.

View on GitHub
/plugin install claude-review-loop

How it works

1

Review

Spawns 4 specialist agents in parallel to examine every change on your branch. Each agent brings a different lens — code quality, silent failures, test coverage, and type design.

2

Triage

Each finding is presented interactively. You decide: approve, modify, defer, or dismiss. You're always in control of what gets fixed.

3

Plan

Enters plan mode and presents a remediation strategy — which files to change, what to change, and why. You review and approve before any code is touched.

4

Fix

Dispatches fixer agents to implement the approved changes. Fixes on different files run in parallel using isolated worktrees. Minimal changes only — no drive-by refactoring.

5

Re-audit

Optionally re-runs the entire review loop to verify fixes and catch regressions. Repeat until clean.

Why use it

👁

Fresh eyes on every change

Adversarial reviewers examine your code with zero assumptions. They catch what you've gone blind to after hours of coding.

⚡

Parallel specialist agents

Four focused agents run simultaneously — each specializing in a different class of defect. Faster and more thorough than a single pass.

🛡

Human in the loop

Every finding goes through you. Nothing gets fixed without your explicit approval. Modify suggestions, defer issues, or dismiss false positives.

🔁

Review, fix, repeat

After fixes are applied, re-run the loop to verify. Iterative refinement until the code is clean. State is preserved across sessions.

📋

Plan before you fix

Claude enters plan mode and presents its remediation strategy for your approval. You see exactly what will change before a single line is edited.

📦

Zero dependencies

Pure markdown skill — no Node.js, no build step, no npm install. Works immediately with just Claude Code.

Specialist agents

The reviewer delegates to four agents from Claude's PR Review Toolkit, running in parallel:

code-reviewer

Bugs, logic errors, null handling, race conditions, security issues, and adherence to project guidelines.

silent-failure-hunter

Empty catch blocks, swallowed errors, missing error propagation, and inadequate error handling.

pr-test-analyzer

Critical untested paths, missing edge case coverage, and gaps in error handling tests.

type-design-analyzer

Invariant strength, encapsulation quality, discriminated union design, and type safety issues.

Get started

Install the plugin:

# From the Claude Code CLI /plugin install claude-review-loop

Enable parallel agent teams in ~/.claude/settings.json:

{ "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }

Add required permissions to ~/.claude/settings.json:

{ "permissions": { "allow": [ "Bash(git *)", "Bash(date *)", "Read(*)", "Write(*)", "Edit(*)", "Glob(*)" ] } }

Then, from any branch with changes:

# Invoke the skill /review-loop
